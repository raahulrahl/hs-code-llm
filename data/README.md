# data/

The dataset itself is generated by [`shelley-data-ingest/us-ingest`](../../shelley-data-ingest/us-ingest/)
out of the live Postgres `bindu_db` (HTSUS schedule + CROSS rulings).
The output snapshot directory is symlinked here as `current/`:

```
data/
└── current/  ─►  ../../regulations/dataset/us/<YYYY-MM-DD>/
    ├── train.jsonl   chat-format SFT input
    ├── eval.jsonl    chat-format held-out (stratified by chapter)
    ├── flat.jsonl    legacy (prompt, completion, code, chapter, source)
    ├── stats.json    per-chapter + per-source counts
    └── README.md     dataset card
```

## To refresh

```bash
cd ../../shelley-data-ingest/us-ingest

# Re-pull CROSS metadata if you want the latest rulings
DATABASE_URL=... .venv/bin/python -m src.cross.run sweep
DATABASE_URL=... .venv/bin/python -m src.cross.run ingest

# (Optional) fetch full ruling text for richer reasoning
DATABASE_URL=... .venv/bin/python -m src.cross.run text --sample-size 5000
DATABASE_URL=... .venv/bin/python -m src.cross.run ingest

# Rebuild the dataset JSONL
DATABASE_URL=... .venv/bin/python -m src.dataset.build

# Repoint the symlink at the latest snapshot
cd -
ln -snf ../../regulations/dataset/us/$(date +%F) data/current
```

The JSONL files themselves are NOT committed — see `.gitignore`. The
canonical source of truth is Postgres; the JSONL is a derived artifact.
